## 1 引言

### 1.1 编写目的

本文档旨在完整描述新闻公告管理系统的功能需求与非功能需求，为后续的设计、开发、测试与验收提供统一依据。目标读者包括开发人员、测试人员、项目评审教师及相关利益方。

### 1.2 项目背景

学校及企业内部长期依赖纸质通知与线下张贴方式发布公告，存在传播效率低、管理不便、存档困难等问题。为改进这一现状，拟搭建一套简易的新闻公告在线管理平台，实现公告的数字化发布与管理，替代传统纸质通知，使管理员可在线发布、编辑、管理公告内容，普通用户可在线浏览公告信息。

本系统定位为简易 CRUD 项目，复杂度与图书管理系统持平，适合作为课程设计或小型实用工具。

### 1.3 项目目标

1. **公告全生命周期管理**：实现公告信息的发布、查看、修改、删除。  
2. **栏目分类管理**：支持公告栏目的新增、修改、删除与查询。  
3. **角色权限分离**：管理员登录后可进行全部增删改查操作；游客无需登录，仅可浏览新闻公告。  
4. **数据分页与检索**：支持按标题模糊搜索、按栏目筛选、按时间筛选，数据以分页形式展示。

### 1.4 适用范围

本系统适用于学校、企业等组织内部公告发布场景，数据规模为单表百级记录，并发用户数不超过 20 人。

### 1.5 术语与缩略语

| 术语 | 说明 |
|------|------|
| 管理员 | 系统中唯一具备增删改查权限的角色，需登录后操作 |
| 游客 | 无需登录即可浏览公告的普通用户角色 |
| 栏目 | 新闻公告的分类，如通知公告、校园新闻等 |
| CRUD | 创建(Create)、读取(Read)、更新(Update)、删除(Delete) |
| SRS | 软件需求规格说明书 (Software Requirements Specification) |

## 2 系统概述

### 2.1 产品描述

新闻公告管理系统采用 B/S 架构，基于 Django 框架开发，MySQL 作为数据持久化存储。系统分为前台浏览端与后台管理端：前台面向前台游客，提供公告列表浏览与详情查看功能；后台面向管理员，提供栏目管理与公告管理的全部增删改查操作。管理后台受登录保护，未认证用户访问管理页面时自动跳转至登录页。

### 2.2 用户角色

| 角色 | 权限范围 | 认证方式 |
|------|----------|----------|
| **管理员** | 登录系统；管理栏目（增删改查）；管理公告（增删改查，含批量删除）；退出登录 | 账号密码登录 |
| **游客** | 浏览新闻公告列表；查看公告详情；按栏目筛选公告 | 无需登录 |

### 2.3 运行环境

| 层次 | 技术选型 | 备注 |
|------|----------|------|
| 后端框架 | Python Django | 提供 MVC 架构、ORM、模板渲染与会话管理 |
| 数据库 | MySQL | 关系型数据库，存储管理员、栏目、公告数据 |
| 前端 | Django Template + 原生 HTML/CSS/JS | 不引入前端框架，界面简洁 |
| 浏览器 | Chrome、Firefox、Edge 等主流浏览器 | 兼容最新两个大版本 |

### 2.4 约束与假设

| 编号 | 约束项 | 说明 |
|------|--------|------|
| C-01 | 无富文本 | 公告内容仅支持纯文本输入，不做富文本编辑器 |
| C-02 | 无文件上传 | 公告不包含图片、附件等文件上传功能 |
| C-03 | 无复杂权限 | 仅区分管理员与游客两类角色，不设角色细分 |
| C-04 | 单管理员 | 系统中预设单一管理员账号，不做多管理员 |
| C-05 | 数据规模 | 公告总数不超过 500 条，栏目不超过 20 个 |
| C-06 | 单机部署 | 系统运行于单台服务器，不考虑分布式 |

## 3 功能需求

### 3.1 登录模块

#### 3.1.1 功能描述

管理员通过登录页面输入账号与密码，系统验证通过后创建会话，允许进入后台管理页面。验证失败时给出明确错误提示。管理员可主动退出登录。

#### 3.1.2 用例

| 编号 | 用例名称 | 前置条件 | 基本事件流 | 后置条件 |
|------|----------|----------|------------|----------|
| UC-LOGIN-01 | 管理员登录成功 | 管理员处于登录页面 | 1. 输入正确账号密码 2. 点击登录 3. 系统验证通过，跳转至栏目管理页 | 会话创建，管理员登录态生效 |
| UC-LOGIN-02 | 管理员登录失败 | 管理员处于登录页面 | 1. 输入错误账号或密码 2. 点击登录 3. 系统提示"账号或密码错误" | 停留在登录页，不清空已输入内容 |
| UC-LOGIN-03 | 管理员退出登录 | 管理员已登录 | 1. 点击退出按钮 2. 系统清空会话 3. 跳转至前台首页 | 会话销毁，管理权限失效 |

#### 3.1.3 界面交互

- 登录表单包含：账号输入框、密码输入框、登录按钮。
- 验证失败后，页面顶部显示红色提示文字，内容为"账号或密码错误"。
- 登录成功后跳转至栏目分类管理页面。

#### 3.1.4 权限校验

- 所有后台管理页面（`/admin/*`）必须验证登录会话。
- 未登录用户访问任何管理页面时，服务端返回 302 重定向到登录页。
- 登录成功后，跳转回用户原本访问的目标页面。

### 3.2 栏目分类管理

#### 3.2.1 功能描述

栏目是新闻公告的分类维度。管理员可以新增栏目、修改栏目名称、删除栏目、查看栏目列表。删除栏目前需确保该栏目下无关联新闻公告，否则拒绝删除。

#### 3.2.2 用例

| 编号 | 用例名称 | 前置条件 | 基本事件流 | 后置条件 |
|------|----------|----------|------------|----------|
| UC-TYPE-01 | 新增栏目 | 管理员已登录，处于栏目管理页 | 1. 点击新增按钮 2. 弹窗中输入栏目名称 3. 点击保存 4. 系统校验名称非空，保存入库 | 栏目列表新增一条记录 |
| UC-TYPE-02 | 新增栏目——名称为空 | 管理员已登录 | 1. 点击新增 2. 弹窗中留空栏目名称 3. 点击保存 4. 系统提示"栏目名称不能为空" | 弹窗不关闭，数据不保存 |
| UC-TYPE-03 | 修改栏目名称 | 管理员已登录 | 1. 点击某栏目的编辑按钮 2. 弹窗中修改名称 3. 点击保存 | 栏目名称更新 |
| UC-TYPE-04 | 删除栏目（无关联新闻） | 管理员已登录，目标栏目下无新闻 | 1. 点击删除按钮 2. 系统二次确认"确定删除该栏目？" 3. 确认后删除 | 栏目从列表中移除 |
| UC-TYPE-05 | 删除栏目（有关联新闻） | 管理员已登录，目标栏目下存在新闻 | 1. 点击删除按钮 2. 系统提示"该栏目下存在新闻，无法删除" | 栏目保留，操作被拒绝 |
| UC-TYPE-06 | 查看栏目列表 | 管理员已登录 | 1. 进入栏目管理页 2. 系统展示所有栏目，以表格形式列出 | 列表包含序号、栏目名称、操作按钮 |

#### 3.2.3 业务规则

- 栏目名称不可为空，不可与已有栏目重名。
- 栏目下存在至少一条新闻公告时，不允许删除该栏目。
- 删除操作须有二次确认弹窗。

### 3.3 新闻公告信息管理

#### 3.3.1 功能描述

新闻公告是本系统的核心数据实体。管理员在后台可新增、修改、删除（含批量）、查询新闻公告，字段包括标题、正文内容、发布人、发布时间、所属栏目。

#### 3.3.2 用例

| 编号 | 用例名称 | 前置条件 | 基本事件流 | 后置条件 |
|------|----------|----------|------------|----------|
| UC-NEWS-01 | 新增公告 | 管理员已登录，至少存在一个栏目 | 1. 点击新增按钮 2. 弹窗填写标题、内容、发布人、选择栏目 3. 点击保存 4. 系统校验标题非空，发布时间自动填充当前时间 | 公告列表新增一条记录 |
| UC-NEWS-02 | 新增公告——标题为空 | 管理员已登录 | 1. 点击新增 2. 弹窗中标题留空 3. 点击保存 4. 系统提示"标题不能为空" | 弹窗不关闭，数据不保存 |
| UC-NEWS-03 | 修改公告 | 管理员已登录 | 1. 点击某公告的编辑按钮 2. 弹窗中修改任意字段 3. 点击保存 | 公告内容更新 |
| UC-NEWS-04 | 单条删除公告 | 管理员已登录 | 1. 点击某公告的删除按钮 2. 系统二次确认 3. 确认后删除 | 公告从列表中移除 |
| UC-NEWS-05 | 批量删除公告 | 管理员已登录，列表至少勾选两条 | 1. 勾选多条公告的复选框 2. 点击批量删除按钮 3. 系统二次确认"确定删除选中的 N 条公告？" 4. 确认后删除 | 被选中的公告全部移除 |
| UC-NEWS-06 | 按标题模糊搜索 | 管理员已登录 | 1. 在搜索框输入关键词 2. 点击搜索/回车 3. 系统返回标题包含关键词的公告，分页展示 | 列表过滤 |
| UC-NEWS-07 | 按栏目筛选 | 管理员已登录 | 1. 在下拉框中选择某一栏目 2. 点击筛选 3. 系统返回该栏目下的公告列表 | 列表过滤 |
| UC-NEWS-08 | 按发布时间筛选 | 管理员已登录 | 1. 在日期选择器中选定日期范围 2. 点击筛选 3. 系统返回该时间段内的公告列表 | 列表过滤 |
| UC-NEWS-09 | 分页展示 | 管理员已登录 | 1. 公告总数超过每页上限（如 10 条） 2. 列表底部显示页码导航 3. 点击页码跳转对应页面 | 按页码展示对应数据 |
| UC-NEWS-10 | 查看公告详情 | 管理员或游客 | 1. 在公告列表点击标题 2. 系统展示公告完整内容 | 详情页显示标题、内容、发布人、发布时间、栏目 |

#### 3.3.3 业务规则

- **标题**：必填，不超过 200 个字符。
- **内容**：选填，纯文本，不超过 10000 个字符。
- **发布人**：选填，不超过 50 个字符。
- **发布时间**：系统自动填充为当前服务器时间，管理员不可手动修改（新增时自动赋值）。
- **所属栏目**：必选，下拉框展示所有已有栏目。
- 批量删除前需二次确认，提示被选中公告的数量。
- 搜索与筛选可组合使用（条件叠加）。

### 3.4 前台浏览模块

#### 3.4.1 功能描述

前台首页面向游客，展示所有已发布的新闻公告列表，按发布时间倒序排列，支持按栏目筛选。游客可点击公告标题进入详情页查看完整内容。

#### 3.4.2 用例

| 编号 | 用例名称 | 前置条件 | 基本事件流 | 后置条件 |
|------|----------|----------|------------|----------|
| UC-FRONT-01 | 浏览公告列表 | 游客进入前台首页 | 1. 页面展示公告列表，按发布时间倒序 2. 每条显示标题、发布人、发布时间、栏目 | 列表支持分页 |
| UC-FRONT-02 | 查看公告详情 | 游客在前台列表页 | 1. 点击某条公告标题 2. 跳转至详情页 3. 显示标题、内容、发布人、发布时间、栏目 | 可点击返回列表 |
| UC-FRONT-03 | 按栏目筛选 | 游客在前台首页 | 1. 在栏目导航/下拉框中选择栏目 2. 列表更新为该栏目下的公告 | 筛选条件可见 |

#### 3.4.3 界面约束

- 前台页面不显示任何编辑、删除、新增按钮。
- 前台页面不包含登录入口（管理后台入口可放置在页面底部或通过 `/admin` 直接访问）。

### 3.5 数据基础功能

| 编号 | 功能 | 说明 |
|------|------|------|
| F-BASE-01 | 分页 | 每页默认显示 10 条记录，带页码导航，支持上一页/下一页 |

## 5 数据需求

### 5.1 数据库环境

- **数据库管理系统**：MySQL 5.7 及以上 / MySQL 8.0
- **字符集**：utf8mb4
- **ORM 框架**：Django ORM

### 5.2 逻辑数据模型

系统包含三张表，关系如下：

```
admin (管理员表)
  ├── id (PK)
  ├── username
  └── password

news_type (栏目表)
  ├── id (PK)
  └── type_name

news (新闻公告表)
  ├── id (PK)
  ├── title
  ├── content
  ├── author
  ├── publish_time
  └── type_id (FK → news_type.id)
```

- `news.type_id` 外键关联 `news_type.id`，删除栏目时若存在关联新闻则拒绝删除（RESTRICT 策略）。

### 5.3 数据字典

#### 5.3.1 管理员表（admin）

| 字段名 | 数据类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|----------|------|------|--------|------|
| id | INT | — | 是 | 自增 | 主键 |
| username | VARCHAR | 50 | 是 | — | 登录账号，唯一 |
| password | VARCHAR | 128 | 是 | — | 登录密码（加密存储） |

#### 5.3.2 新闻栏目表（news_type）

| 字段名 | 数据类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|----------|------|------|--------|------|
| id | INT | — | 是 | 自增 | 主键 |
| type_name | VARCHAR | 50 | 是 | — | 栏目名称，不可重复 |

#### 5.3.3 新闻公告表（news）

| 字段名 | 数据类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|----------|------|------|--------|------|
| id | INT | — | 是 | 自增 | 主键 |
| title | VARCHAR | 200 | 是 | — | 公告标题 |
| content | TEXT | — | 否 | NULL | 公告正文内容 |
| author | VARCHAR | 50 | 否 | NULL | 发布人 |
| publish_time | DATETIME | — | 是 | CURRENT_TIMESTAMP | 发布时间，自动填充 |
| type_id | INT | — | 是 | — | 所属栏目 ID，外键关联 news_type.id |

#### 5.3.4 索引建议

| 索引名 | 表 | 字段 | 类型 |
|--------|-----|------|------|
| idx_news_type_id | news | type_id | 普通索引 |
| idx_news_publish_time | news | publish_time | 普通索引（降序） |
| idx_news_title | news | title | 普通索引（用于模糊搜索） |

- "返回列表"按钮或链接。

## 7 业务流程

### 7.1 管理员操作流程

```
[打开后台登录页] → [输入账号密码] → {验证}
                                      ├─ 失败 → [提示错误] → [重新输入]
                                      └─ 成功 → [进入栏目管理页]
                                                   │
                                                   ├─ [维护栏目]
                                                   │   ├─ 新增栏目
                                                   │   ├─ 修改栏目名称
                                                   │   └─ 删除栏目（无关联新闻时允许）
                                                   │
                                                   ├─ [进入公告管理]
                                                   │   ├─ 新增公告（填写标题/内容/发布人/选择栏目）
                                                   │   ├─ 修改公告
                                                   │   ├─ 删除公告（单条/批量）
                                                   │   └─ 搜索/筛选公告
                                                   │
                                                   └─ [退出登录] → [清空会话] → [返回前台首页]
```

### 7.2 游客浏览流程

```
[打开前台首页] → [浏览公告列表（按时间倒序）]
                   │
                   ├─ [按栏目筛选] → [查看该栏目公告]
                   │
                   └─ [点击标题] → [查看公告详情] → [返回列表]
```

### 7.3 系统流程约束

1. 管理员必须先维护栏目，再创建公告（公告必须关联已存在的栏目）。
2. 删除栏目前，系统自动检查该栏目下是否存在关联公告。
3. 未登录时访问任何 `/admin/*` 路径，系统无条件重定向至登录页。
4. 登录态到期或退出后，再次访问管理页面需重新认证。
